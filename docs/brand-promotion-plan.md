# 「一键做企宣」完整功能方案

## 用户场景

老板/商家上传产品照片或视频 → AI 自动生成多语言宣传视频 → 可选数字人出镜讲解

## 功能流程

```
┌─────────────┐   ┌──────────────┐   ┌─────────────────┐   ┌──────────┐
│ 上传素材      │ → │ AI 生成文案   │ → │ 多语言配音+数字人  │ → │ 预览导出  │
│ 产品照片/视频 │   │ 自动写卖点文案 │   │ 多国语言TTS      │   │ 下载MP4  │
│ 企业logo    │   │ 多语言翻译   │   │ 数字人出镜讲解   │   │ 分享链接  │
└─────────────┘   └──────────────┘   └─────────────────┘   └──────────┘
```

## 页面设计

**页面路径：** `/brand-promotion`（导航"一键做企宣"已指向 `/ai-video`，改为指向这个新页面）

### 布局（三步向导式）

#### Step 1：上传素材
- 上传产品照片（支持多张）
- 上传产品视频（可选）
- 上传企业 Logo（可选）
- 输入产品名称/卖点（可选，不填则 AI 自动识别）

#### Step 2：配置参数
- **宣传语风格**：专业商务 / 社交热辣 / 科技感 / 朴实真诚
- **目标语言**：中文、英语、日语、韩语、阿拉伯语（多选）
- **数字人开关**：开/关
  - 如开启：选择数字人形象（男/女/现有数字人）
  - 是否手持产品讲解
- **视频长度**：15秒 / 30秒 / 60秒

#### Step 3：生成 + 预览
- 显示生成进度（文案生成 → 配音 → 视频合成 → 数字人）
- 预览视频
- 切换不同语言版本预览
- 下载 MP4

## 技术架构

### 前端
- 新页面 `/app/brand-promotion/page.tsx` — 三步向导
- 组件拆分：
  - `UploadZone` — 拖拽上传
  - `ConfigPanel` — 参数配置
  - `LanguageSelector` — 多语言选择
  - `ProgressView` — 生成进度
  - `PreviewPlayer` — 视频预览
  - `DigitalHumanToggle` — 数字人开关

### 后端 API
| API | 用途 | 依赖 |
|-----|------|------|
| `POST /api/brand-promotion/upload` | 上传素材 | 文件系统 |
| `POST /api/brand-promotion/script` | AI 生成卖点文案 | OpenAI |
| `POST /api/brand-promotion/translate` | 多语言翻译 | OpenAI |
| `POST /api/brand-promotion/tts` | 多语言配音 | OpenAI TTS |
| `POST /api/brand-promotion/render` | 视频合成渲染 | FFmpeg |
| `POST /api/brand-promotion/digital-human` | 数字人出镜 | Agnes API |
| `GET  /api/brand-promotion/status/:id` | 查询生成进度 | - |

### 已有可复用能力
- ✅ OpenAI API（文案生成、翻译）
- ✅ OpenAI TTS（多语言配音）
- ✅ Agnes API（数字人，已有配置）
- ✅ 服务端 FFmpeg（视频合成，需确认服务器是否安装）
- ❌ 手持产品数字人（需要额外配置或新数字人模板）

## 需要老大确认的点

### 1️⃣ 数字人方案
现有数字人能力（ai-video 页面）是通过 **Agnes API** 生成数字人口播视频。能做"站立口播"但不能"手持产品"。
- **方案A**：先用 Agnes 的站姿口播，产品以画中画/PIP 形式插入（技术可行，本周可上线）
- **方案B**：找专门支持手持产品的数字人服务（如 HeyGen、D-ID），更真实但需要额外对接
- **方案C**：只有配音无数字人，纯产品展示视频（最简单最快）

老大倾向哪个？

### 2️⃣ 多语言优先支持哪些？
我建议：**中、英、日、韩、阿拉伯**（覆盖主要跨境市场）

### 3️⃣ 宣传视频长度
- 15秒（短视频平台）
- 30秒（标准广告）
- 60秒（详细展示）

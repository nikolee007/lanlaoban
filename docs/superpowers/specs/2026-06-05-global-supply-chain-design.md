# 懒老板 2.0：全球供应链模块设计文档

> 日期：2026-06-05
> 产品矩阵：懒老板（IP 打造） + 赚了么（门店盘点） + **全球供应链（资源对接）**
> 核心理念：从用户视角出发，不做自嗨功能

---

## 一、产品定位

**全球供应链 = 老板的一站式资源对接平台**

帮老板找货、找厂、找物流、找渠道，国内国外都覆盖。核心理念是"帮老板省时间、找对路"——不是黄页，而是带认证、带口碑、带选品洞察的智能资源库。

### 用户类型

| 用户 | 场景 | 核心需求 |
|------|------|----------|
| 跨境卖家（国内） | 选品找货、找物流、对接海外渠道 | TikTok 爆款、一件代发、供应链验真 |
| 海外商家（境外） | 找国内工厂、找供应链落地 | 小批量试单、OEM 定制、跨境物流 |
| 国内实体老板 | 找本地供应链、找包装耗材 | 餐饮供应链、门店耗材、本地服务商 |

## 三、数据层架构

### 3.1 技术选型

| 层 | 技术 | 说明 |
|----|------|------|
| 框架 | Next.js 14 (App Router) | 维持现有 |
| 数据库 | Prisma + SQLite | MVP 先行，可平滑升 PostgreSQL |
| 数据采集 | Python scripts (公开数据抓取) | 独立脚本，不阻塞前端 |
| AI 集成 | 已有 OpenAI 能力复用 | 生成视频脚本/选品洞察 |

### 3.2 数据模型

```
Supplier (供应商/工厂)
├── id, name, nameEn, logo
├── location (province, city, address)
├── establishedYear, employeeCount, annualExport
├── certifications[] (ISO, FCC, CE...)
├── mainBusiness[] (主营产品标签)
├── exportDestinations[] (出口地区)
├── avgRating, reviewCount
├── verified (懒老板认证)
├── lastUpdated (数据更新时间)
└── type: factory | brand | distributor

Product (商品)
├── id, name, description
├── supplierId → Supplier
├── categoryId → Category
├── priceRange (min, max, currency)
├── moq (起订量)
├── supportDropship (一件代发)
├── supportOEM
├── images[]
├── certifications[]
├── salesData (TikTok/Amazon 销售参考)
├── avgRating, reviewCount
└── lastUpdated

AggregatedReview (聚合口碑)
├── id
├── targetId, targetType (supplier|product)
├── platform (1688|Amazon|TikTokShop)
├── avgRating, reviewCount
├── returnRate, repurchaseRate
├── topKeywords[] (评价关键词云)
└── collectedAt (采集时间)

LogisticsProvider (物流服务商)
├── id, name, logo
├── routes[] (航线/线路)
├── priceRange
├── avgTransitDays
├── coverage[] (覆盖国家/地区)
├── avgRating
└── lastUpdated

Channel (销售渠道)
├── id, name (TikTok Shop, Amazon, Shopee...)
├── type (platform|distributor|retailer)
├── coverage[] (覆盖地区)
├── requirements (入驻要求)
└── lastUpdated

UserCollection (用户收藏)
├── id, userId
├── targetId, targetType
├── notes (用户备注)
└── createdAt

Category (分类/行业)
├── id, name, parentId
├── icon, sortOrder
└── type (domestic|crossborder)
```

## 四、六项优化落地方案

### 4.1 小B模式（一件代发/小批量）

- **数据层**：Product 模型增加 `supportDropship: boolean`、`moqMin`、`moqMax` 字段
- **搜索层**：筛选器增加"一件代发"选项
- **UI 层**：卡片显示 📦 一件代发: ✅ | 最小起订: N 件
- **数据来源**：公开信息 + 后续商家自主提交

### 4.2 商品 ↔ AI 视频素材打通

- **入口**：商品详情页和搜索结果卡片都放「🎬 生成视频素材」按钮
- **流程**：点击 → 弹窗输入品牌名/卖点 → 调用已有 AI 能力 → 生成脚本/口播 → 用户下载
- **技术**：复用 `app/api/generate/scripts/` 和 `app/api/digital-human/` 现有接口
- **MVP 版本**：生成短视频脚本 + 口播文案（数字人生成留到后续）

### 4.3 国内供应链板块

- **分类层**：Category 增加 `type: domestic | crossborder | both`
- **首页层**：行业分类卡片同时展示国内和跨境分类
- **数据层**：优先填充国内公开数据（1688 公开信息、企业黄页等）
- **差异化**：国内板块突出"本地货源、餐饮供应链、门店耗材"等实体老板刚需

### 4.4 每周上新/更新机制

- **首页**：上新速递板块 "本周新入驻 N 家工厂"
- **勋章**：新入驻企业 7 天内显示 "新" 角标
- **自动化**：数据采集脚本每周自动抓一轮公开数据
- **Pull 机制**：用户可关注品类/行业，上新时首页置顶

### 4.5 数据新鲜度管理

- **每个卡片**显示 `🕐 数据更新: YYYY-MM-DD`
- **每个详情页**显示各板块数据采集时间
- **过期预警**：超过 30 天未更新的数据标注"数据较旧，仅供参考"
- **自动刷新**：采集脚本定期更新口碑/价格数据

### 4.6 选品洞察

- **搜索结果页**：懒老板选品洞察卡片，告诉老板"为什么这个品类好卖"
- **数据来源**：AI 分析公开趋势数据 + TikTok 热门品类信息
- **格式**：一句话 + 数据支撑（如"蓝牙耳机在东南亚 TikTok 搜索量月度增长 40%"）
- **首页可选**：洞察板块在首页轮播展示
